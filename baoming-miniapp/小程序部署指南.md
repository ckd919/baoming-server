# 🐻 小棕熊报名工具 - H5 转微信小程序完整指南

> 适合零基础新手，每一步都有截图说明

---

## 📋 前置准备（一次性）

### 1. 注册微信小程序账号

1. 打开 https://mp.weixin.qq.com → 点击右上角「立即注册」
2. 选择「小程序」类型
3. 填写邮箱、密码（**不能用已注册公众号的邮箱**）
4. 邮箱激活 → 选择「个人」主体 → 填写身份证 + 手机号
5. 管理员微信扫码验证
6. 注册完成后，登录 → 左侧菜单「开发」→「开发管理」→「开发设置」
7. 记下 **AppID(小程序ID)**，后面要用

### 2. 下载安装 HBuilderX

1. 下载：https://www.dcloud.io/hbuilderx.html
2. 选 **App开发版**（Windows 下载 .exe）
3. 安装后打开

### 3. 下载微信开发者工具

1. 下载：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 选 **Windows 64** 稳定版
3. 安装（一路下一步）

---

## 🚀 导入项目到 HBuilderX

1. 打开 HBuilderX
2. 文件 → 导入 → 从本地目录导入
3. 选择 `d:\claudeCode\baoming\baoming-miniapp\`
4. 项目导入成功

### 修改 AppID 和域名

打开 `manifest.json`，找到：

```json
"mp-weixin": {
  "appid": "请替换为你的小程序AppID",
}
```

把 `请替换为你的小程序AppID` 替换成你自己的 AppID。

打开 `store/api.js`，修改：

```javascript
const API_BASE = 'https://你的域名.com/api'
```

替换成你的后端域名（如 `http://8.134.252.128/api`）。

## ▶️ 本地运行调试

1. HBuilderX 顶栏 → 运行 → 运行到小程序模拟器 → 微信开发者工具
2. 第一次会让你选择微信开发者工具安装路径：
   - 通常在这里：`C:\Program Files (x86)\Tencent\微信web开发者工具`
3. 微信开发者工具会自动打开，显示小程序界面

### 真机预览

微信开发者工具顶部 → 点击「预览」→ 生成二维码 → 手机微信扫码即可在手机上预览。

## 🌐 后端配置（重要）

### 小程序要求 HTTPS

微信小程序**只允许访问 HTTPS 域名**，开发阶段可以临时关闭验证：

微信开发者工具 → 右上角「详情」→ 本地设置 → **勾选「不校验合法域名」**

### 上线前必须配置 HTTPS

1. 给你的域名配置 SSL 证书（Let's Encrypt 免费）：
```bash
yum install -y certbot python3-certbot-nginx
certbot --nginx -d 你的域名.com
```

2. 在微信小程序后台 → 开发 → 开发管理 → 服务器域名：
   - **request合法域名** 添加：`https://你的域名.com`

## 📤 上传发布

### 1. 上传代码

HBuilderX → 发行 → 小程序-微信（仅适用于uni-app）→ 填入 AppID → 发行

等待编译完成，会自动打开微信开发者工具。

### 2. 提交审核

微信开发者工具 → 右上角「上传」→ 填写版本号和备注 → 上传

### 3. 提交审核

1. 登录 https://mp.weixin.qq.com
2. 左侧「管理」→「版本管理」
3. 开发版本 → 点击「提交审核」
4. 填写审核信息（功能页面截图、测试账号等）
5. 提交后等待审核（1-7个工作日）

### 4. 审核通过后发布

审核通过 → 版本管理 → 「发布」→ 全量发布

## 🔧 常见问题

### Q: HBuilderX 运行找不到微信开发者工具
**A:** 工具 → 设置 → 运行配置 → 手动指定「微信开发者工具路径」

### Q: 报错 "不在以下 request 合法域名列表中"
**A:** 开发工具里勾选「不校验合法域名」，上线后到小程序后台配置

### Q: TabBar 图标不显示
**A:** 把图标放到 `static/` 目录下（需要 40x40 像素的 PNG）

### Q: 登录失败 / 网络请求失败
**A:** 检查 `store/api.js` 中的 `API_BASE` 是否正确，后端服务是否运行

### Q: 小程序审核不通过
**A:** 常见原因：
- 缺少隐私政策页面
- 类目选择不对（选「工具-信息查询」）
- 功能不完整（需要提交测试账号）

## 💡 技术栈对照

| H5版本 | 小程序版本 |
|--------|-----------|
| Vite + Vanilla JS | uni-app (Vue 2) |
| HTML + CSS | WXML + WXSS → uni-app 自动转换 |
| DOM 操作 | Vue 响应式数据绑定 |
| localStorage | uni.getStorageSync |
| fetch API | uni.request |
| Hash路由 | 页面栈导航 |
| Nginx 部署 | 微信云托管 |

后端（Node.js + Express + PostgreSQL）**完全不需要改动**，H5 和小程序共用同一套 API。
