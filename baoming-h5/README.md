## 🚀 快速开始（新手友好）

### 第一步：安装 Node.js

如果你还没有 Node.js，请先安装：

1. 打开 [https://nodejs.org](https://nodejs.org)
2. 下载 **LTS 版本**（推荐 18.x 或 20.x）
3. 双击安装，一路点"下一步"即可
4. 安装完成后，打开 **命令提示符**（Win+R 输入 `cmd`）验证：

```bash
node --version    # 应显示 v18.x.x 或 v20.x.x
npm --version     # 应显示 9.x.x 或 10.x.x
```

### 第二步：安装项目依赖

在命令提示符中进入项目目录：

```bash
cd d:\claudeCode\baoming\baoming-h5
npm install
```

> ⏳ 等待 1-2 分钟，看到 `added XX packages` 就表示成功了

### 第三步：启动开发服务器

```bash
npm run dev
```

看到以下输出就表示启动成功：

```
VITE v5.x.x  ready in xxx ms
➜  Local:   http://localhost:3000/
➜  Network: use --host to expose
```

浏览器会自动打开 `http://localhost:3000`，你会看到登录页面。

### 第四步：开始体验

1. 在登录页输入手机号 `13310843925`，密码 `Aled2239`
2. 点击登录进入仪表盘
3. 点击右下角 **+** 号创建活动
4. 填写活动信息后进入**表单构建器**（核心功能）
5. 从字段面板点击添加收集字段
6. 点击字段卡片可编辑属性、设置必填、添加选项
7. 保存后去发布页生成报名链接
8. 复制链接在手机或新标签页打开，体验报名流程

## 🔍 如何调试（新手必读）

### 1. 打开开发者工具

在浏览器中按 **F12**（或右键 → 检查），会打开开发者工具面板。

### 2. 查看 Console（控制台）

- 点击 **Console** 标签
- 这里会显示应用的日志和**错误信息**
- 红色信息 = 错误，黄色信息 = 警告
- 应用启动时会输出预设账号信息

### 3. 查看 Network（网络请求）

- 点击 **Network** 标签
- 可以看到所有网络请求
- 本应用是纯前端，没有后端请求，所以 Network 基本是空的

### 4. 查看 Application → Local Storage（本地存储）

- 点击 **Application** 标签
- 左侧找到 **Local Storage** → `http://localhost:3000`
- 可以看到所有存储的数据：
  - `bm_user` - 登录用户信息
  - `bm_activities` - 所有活动数据（JSON 数组）
  - `bm_submissions_xxx` - 每个活动的报名提交数据

**这是强大的调试手段！** 你可以：
- 查看数据结构是否正确
- 手动修改数据测试不同状态
- 删除某个 key 清除数据

### 5. 移动端调试

在电脑浏览器中模拟手机：

1. 按 **F12** 打开开发者工具
2. 点击 **设备切换按钮**（手机图标，或按 `Ctrl+Shift+M`）
3. 选择 iPhone 12 Pro 或任意手机型号
4. 刷新页面即可看到手机效果

### 6. 真机调试

要让手机访问电脑上的页面：

```bash
# 修改 vite.config.js，添加 --host 或修改配置
npx vite --host
```

然后用手机访问终端显示的 Network 地址（如 `http://192.168.x.x:3000`）。

> ⚠️ 手机和电脑必须在同一个 WiFi 下

## 🛠️ 常见问题排查

### 问题：`npm` 不是内部或外部命令

**原因**：Node.js 没有正确安装或没有加入环境变量

**解决**：
1. 重新安装 Node.js（从 nodejs.org 下载 LTS 版本）
2. 安装时确保勾选 "Add to PATH"
3. 安装后**重新打开**命令提示符

### 问题：`npm install` 卡住或报错

**解决**：
1. 检查网络连接
2. 尝试用国内镜像：
   ```bash
   npm config set registry https://registry.npmmirror.com
   npm install
   ```
3. 删除 `node_modules` 文件夹后重试：
   ```bash
   rmdir /s /q node_modules
   npm install
   ```

### 问题：浏览器显示空白页

**排查步骤**：
1. 按 F12 打开控制台，查看是否有红色错误信息
2. 确认 `npm run dev` 正在运行
3. 确认浏览器地址是 `http://localhost:3000`
4. 尝试刷新页面（Ctrl+R）

### 问题：样式显示不正常

**排查步骤**：
1. 按 F12 → Console 查看是否有 CSS 加载错误
2. 按 F12 → Network 查看 `style.css` 是否加载成功（状态码 200）
3. 确认浏览器缩放比例为 100%

### 问题：数据丢失了

**原因**：数据存储在浏览器的 localStorage 中

**可能原因**：
1. 清除了浏览器缓存/历史记录
2. 使用了浏览器的无痕/隐私模式
3. 切换了浏览器

**解决**：在普通模式下使用，避免清除浏览器数据

- **Vite 5** - 构建工具 & 开发服务器

## 🔒 安全说明

- 这是**纯前端项目**，数据存储在浏览器 localStorage
- 密码验证是**模拟的**，生产环境需要后端服务
- 图片以 base64 存储在 localStorage 中（注意存储限制 ~5MB）
- 不要将真实敏感数据存储在此演示版本中
